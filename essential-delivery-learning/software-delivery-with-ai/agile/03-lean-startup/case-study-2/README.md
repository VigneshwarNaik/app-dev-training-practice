# Case Study 2: Bus Travel Booking System

## 📌 Overview

This case study explores how **Agile** is used to develop a modern **Bus Travel Booking System**.

The system allows users to:

* Search buses
* Book tickets
* Make online payments
* Track bus schedules

Agile helps teams deliver features quickly, adapt to changes, and improve customer satisfaction.

---

## 🎯 Learning Objectives

By the end of this case study, you will understand:

* Agile principles and frameworks
* Sprint planning and backlog management
* Team collaboration
* Incremental delivery
* How Agile handles changing requirements

---

# 1. Problem Statement

A bus company wants to build an online booking platform with features such as:

* User Registration
* Search Buses
* Seat Selection
* Ticket Booking
* Online Payment
* Notifications

Building all features at once can be risky because customer needs may change.

---

# 2. Why Agile?

Agile develops software in **small steps** called **Sprints**.

Each sprint delivers working software.

> **Build small → Get feedback → Improve**

---

# 3. Agile Workflow

```text
Requirements
      ↓
Sprint Planning
      ↓
Development
      ↓
Testing
      ↓
Review
      ↓
Release
      ↓
Repeat
```

Each sprint usually lasts **1–4 weeks**.

---

# 4. Sprint Planning

During sprint planning, the team decides what features to build.

## Sprint 1

Build:

```text
✓ User Registration
✓ Login
```

---

## Sprint 2

Build:

```text
✓ Search Buses
✓ Bus Details
```

---

## Sprint 3

Build:

```text
✓ Seat Selection
✓ Ticket Booking
✓ Payment
```

---

## Sprint 4

Build:

```text
✓ Notifications
✓ Booking History
```

After every sprint:

```text
Develop → Test → Review → Release
```

---

# 5. Backlog Management

A **Backlog** is a list of all project features.

## Product Backlog

```text
✓ Search Buses
✓ Seat Selection
✓ Payment
✓ Notifications
✓ Reviews
✓ Loyalty Program
```

The team prioritizes important features first.

### High Priority

```text
✓ Search Buses
✓ Booking
✓ Payment
```

### Low Priority

```text
✓ Reviews
✓ Loyalty Program
```

---

# 6. Team Collaboration

Agile teams work together closely.

A cross-functional team includes:

* Developers
* Testers
* Designers
* Business Analysts
* Product Owner

Good communication leads to better products.

---

# 7. Incremental Delivery

Instead of waiting months for the final product, Agile delivers features gradually.

### Example

After Sprint 1:

```text
Users can login.
```

After Sprint 2:

```text
Users can search buses.
```

After Sprint 3:

```text
Users can book tickets.
```

Customers receive value continuously.

---

# 8. Challenges Addressed

## Changing Customer Requirements

Customers may request:

* UPI payments
* New filters
* Live bus tracking

Agile allows easy adaptation to these changes.

---

## Real-Time Integrations

The system integrates with:

* Payment gateways
* Bus databases
* Notification services

Real-time data ensures accurate schedules and availability.

---

## Improved User Experience

The team continuously improves:

* Search speed
* Mobile responsiveness
* Booking process

This increases customer satisfaction.

---

# 9. Benefits of Agile in Bus Booking Systems

✅ Faster delivery

✅ Continuous customer feedback

✅ Better software quality

✅ Reduced project risk

✅ Flexibility to adapt to changes

✅ Higher customer satisfaction

---

# 🔑 Key Takeaways

* Agile develops software in small iterations called **Sprints**.
* Teams prioritize features using a **Backlog**.
* Customer feedback helps improve the product.
* Agile delivers working software frequently.
* Agile easily handles changing requirements.
