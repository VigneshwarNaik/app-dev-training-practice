# 1. Software Development Life Cycle (SDLC)

## What is SDLC?

The **Software Development Life Cycle (SDLC)** is a structured process used to develop software from idea to deployment and maintenance.

### SDLC Phases

```text
Requirement Gathering
        ↓
Planning
        ↓
Design
        ↓
Development
        ↓
Testing
        ↓
Deployment
        ↓
Maintenance
```

### Example

Building a **Bus Booking System**:

* Gather requirements
* Design the system
* Develop features
* Test the application
* Release to users
* Maintain and improve

---

# 2. Project Characteristics

A project is a temporary effort to create a unique product or service.

## Key Characteristics

* **Unique:** Every project has unique goals.
* **Temporary:** Projects have start and end dates.
* **Defined Scope:** Clear objectives and deliverables.
* **Constraints:** Limited by time, cost, and scope.
* **Risk:** Uncertainties may affect project success.

### Triple Constraint

```text
Time ↔ Cost ↔ Scope
```

Changing one constraint affects the others.

---

# 3. Traditional SDLC (Waterfall Model)

The **Waterfall Model** follows a sequential approach where each phase is completed before moving to the next.

```text
Requirements
     ↓
Design
     ↓
Development
     ↓
Testing
     ↓
Deployment
```

## Advantages

* Easy to understand
* Well documented
* Suitable for fixed requirements

## Disadvantages

* Difficult to accommodate changes
* Testing occurs late
* Customer feedback is delayed

---

# 4. Agile Foundations

## What is Agile?

Agile is an iterative approach to software development that delivers software in small increments called **Sprints**.

### Agile Process

```text
Plan
  ↓
Develop
  ↓
Test
  ↓
Review
  ↓
Release
  ↓
Repeat
```

## Benefits of Agile

* Faster delivery
* Continuous feedback
* Better quality
* Greater flexibility
* Reduced risk

---

# 5. Traditional SDLC vs Agile

| Feature              | Traditional SDLC | Agile      |
| -------------------- | ---------------- | ---------- |
| Approach             | Sequential       | Iterative  |
| Requirement Changes  | Difficult        | Easy       |
| Customer Involvement | Low              | High       |
| Testing              | End of project   | Continuous |
| Delivery             | One-time         | Frequent   |
| Flexibility          | Low              | High       |

---

# Real-World Example

## Flight Booking Application

### Waterfall Approach

1. Gather all requirements.
2. Design the complete system.
3. Develop all features.
4. Test after completion.
5. Release the final product.

### Agile Approach

* Sprint 1: User Login
* Sprint 2: Flight Search
* Sprint 3: Booking and Payment
* Sprint 4: Notifications

Each sprint delivers working software and incorporates customer feedback.

---

# Key Takeaways

* SDLC provides a roadmap for software development.
* Projects have goals, constraints, and risks.
* Waterfall is suitable for stable requirements.
* Agile supports changing requirements and rapid delivery.
* Modern software projects widely adopt Agile methodologies.
