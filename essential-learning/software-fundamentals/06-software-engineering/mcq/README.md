# Question 1

## Question

Which confirms that the software meets its technical specifications?

### Options

### a.
```text
Verification
```

### b.
```text
Validation
```

### c.
```text
Defect finding
```

### d.
```text
Design
```

---

# ✅ Correct Answer

```text
Verification
```

---

# 📖 Detailed Explanation

Verification is the process of checking whether:
- Software is being developed correctly
- Technical specifications are followed
- Development standards are maintained

Verification checks:

```text
"Are we building the product right?"
```

It focuses on:
- Process correctness
- Technical accuracy
- Design correctness

---

# Verification Activities

Examples:
- Reviews
- Walkthroughs
- Inspections
- Static analysis

---

# Verification vs Validation

| Verification | Validation |
|---|---|
| Checks technical correctness | Checks customer satisfaction |
| Process-oriented | Product-oriented |
| Done during development | Done after development |
| Prevents defects | Detects defects |

---

# Why Other Options are Wrong

## Validation ❌
Validation ensures the software meets customer requirements.

It checks:

```text
"Are we building the right product?"
```

---

## Defect Finding ❌
Defect finding is only one testing activity.

---

## Design ❌
Design is a phase in SDLC, not a verification method.

---

# Question 2

## Question

The software is put into operation in the client’s environment. The client later requests UI enhancements to attract more customers.

This phase is called ______.

### Options

### a.
```text
Development
```

### b.
```text
Maintenance
```

---

# ✅ Correct Answer

```text
Maintenance
```

---

# 📖 Detailed Explanation

Maintenance phase begins:
- After software deployment
- When changes or updates are required

Maintenance ensures:
- Software continues functioning properly
- User requirements are updated
- Improvements are added

---

# Activities in Maintenance Phase

Examples:
- Bug fixing
- UI enhancement
- Security updates
- Performance optimization
- Feature addition

---

# Types of Maintenance

## 1. Corrective Maintenance
Fixes defects and bugs.

---

## 2. Adaptive Maintenance
Updates software according to environment changes.

---

## 3. Perfective Maintenance
Improves features and user interface.

---

## 4. Preventive Maintenance
Improves maintainability for future use.

---

# Why Development is Wrong ❌

Development phase includes:
- Coding
- Implementation
- Initial software creation

The scenario clearly states:
> Changes requested after deployment

So it belongs to the maintenance phase.

# Question 3

## Question

Match the appropriate SDLC process model based on the nature of requirements.

| Scenario |
|---|
| When the requirements are unstable |
| When the requirements are unclear |

---

# ✅ Correct Answers

| Scenario | Correct Model |
|---|---|
| When the requirements are unstable | Spiral Model |
| When the requirements are unclear | Prototype Model |

---

# 📖 Detailed Explanation

# 1. Spiral Model

Spiral Model is suitable when:
- Requirements change frequently
- Risks are high
- Project complexity is high

It focuses heavily on:
- Risk analysis
- Continuous evaluation
- Iterative development

---

# Features of Spiral Model

- Risk-driven model
- Customer feedback after every cycle
- Multiple development iterations
- Combines waterfall and prototyping concepts

---

# Why Spiral Model Handles Unstable Requirements

Because:
- Requirements can evolve gradually
- Every cycle allows modifications
- Risks are evaluated continuously

---

# 2. Prototype Model

Prototype Model is suitable when:
- Requirements are unclear
- Users cannot explain exact needs initially
- Frequent customer interaction is needed

A prototype helps:
- Visualize the system
- Improve communication
- Clarify misunderstandings

---

# Features of Prototype Model

- Early working model created
- Continuous user feedback
- Rapid requirement understanding
- Incremental refinement

---

# Comparison Between Spiral and Prototype Models

| Spiral Model | Prototype Model |
|---|---|
| Focuses on risk management | Focuses on requirement clarification |
| Suitable for high-risk systems | Suitable for unclear requirements |
| Complex projects | UI-intensive projects |
| Multiple development loops | Prototype refinement cycles |

---

# Question 4

## Question

Which of the following is one phase of the Prototype Model?

### Options

### a.
```text
Extensive testing
```

### b.
```text
Detailed Coding
```

### c.
```text
Quick design
```

### d.
```text
Detailed HLD phase
```

---

# ✅ Correct Answer

```text
Quick design
```

---

# 📖 Detailed Explanation

Prototype Model follows these stages:

1. Requirement gathering
2. Quick design
3. Build prototype
4. User evaluation
5. Prototype refinement
6. Final implementation

---

# What is Quick Design?

Quick design:
- Creates rough system design
- Focuses on visible features
- Helps rapidly develop a prototype

Purpose:
- Quickly demonstrate the concept
- Gather early customer feedback

---

# Importance of Quick Design

It helps:
- Reduce misunderstanding
- Clarify requirements
- Improve communication
- Save development time

---

# Why Other Options are Wrong

## Extensive Testing ❌
Testing occurs later after development.

---

## Detailed Coding ❌
Prototype model initially emphasizes rapid development rather than detailed coding.

---

## Detailed HLD Phase ❌
Prototype model uses quick design rather than extensive documentation initially.
