# 🚀 Software Development Life Cycle (SDLC)

## 📌 Overview

This repository contains my learning notes and practical understanding of the Software Development Life Cycle (SDLC). SDLC is a structured process used to design, develop, test, deploy, and maintain high-quality software.

---

## 🤔 What is SDLC?

Software Development Life Cycle (SDLC) is a systematic approach followed by software teams to develop software efficiently and reliably.

### Why is SDLC Important?

* Ensures high-quality software
* Reduces development cost
* Improves project management
* Helps deliver software on time
* Simplifies maintenance and updates

---

## 🔄 SDLC Phases

```text
Analyze → Plan → Build → Test → Deploy → Maintain
```

---

## 1️⃣ Analyze Phase

The Analyze phase focuses on understanding business and user requirements.

### Activities:

* Gather requirements
* Identify project goals
* Understand user needs
* Create Software Requirement Specification (SRS)

### Example:

For a food delivery app:

* User login
* Search restaurants
* Place orders
* Online payment

---

## 2️⃣ Plan Phase

The planning phase defines the project roadmap.

### Activities:

* Estimate budget
* Allocate resources
* Create timelines
* Identify risks

### Team Members:

* Project Manager
* Developers
* Testers
* Business Analysts

---

## 3️⃣ Build Phase

Developers write code and build the software.

### Activities:

* Design database
* Develop frontend
* Develop backend
* Integrate modules

### Technologies:

* Java
* Python
* MySQL
* React

---

## 4️⃣ Test Phase

Testing ensures the software works correctly.

### Types of Testing:

* Functional Testing
* Performance Testing
* Security Testing

### Objective:

Identify and fix defects before release.

---

## 5️⃣ Deploy Phase

Deployment releases the software to end users.

### Examples:

* Google Play Store
* Cloud Servers
* Web Hosting Platforms

---

## 6️⃣ Maintenance Phase

Maintenance supports software after deployment.

### Activities:

* Bug fixes
* Performance improvements
* Security updates
* Feature enhancements

---

## 🏦 Real-World Example: Banking Application

| SDLC Phase | Example                                |
| ---------- | -------------------------------------- |
| Analyze    | Gather requirements for money transfer |
| Plan       | Define budget and schedule             |
| Build      | Develop banking features               |
| Test       | Verify transactions                    |
| Deploy     | Release application                    |
| Maintain   | Add new services and fix bugs          |

---

## ✅ Advantages of SDLC

* Better software quality
* Reduced costs
* Faster delivery
* Improved collaboration
* Easier maintenance
