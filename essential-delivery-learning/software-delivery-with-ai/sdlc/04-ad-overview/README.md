At the end of this module, you will be able to:

---

# 🚀 What is Application Development (AD)?

**Application Development (AD)** is the process of creating software applications to solve user problems and meet business needs.

### Simple Definition

> Application Development is the process of designing, building, testing, and maintaining software applications.

### Examples

* Messaging App 📱
* Banking App 🏦
* Food Delivery App 🍔
* E-commerce App 🛒

---

# 🔄 Important Processes in an AD Project

An Application Development project typically follows these stages:

1. Requirement Gathering
2. Design
3. Development
4. Peer Review
5. Testing
6. Deployment
7. Maintenance

### Workflow

```text
Requirement Gathering
         ↓
      Design
         ↓
    Development
         ↓
    Peer Review
         ↓
      Testing
         ↓
    Deployment
         ↓
    Maintenance
```

---

# 1️⃣ Requirement

## What is Requirement?

Requirements define **what the customer wants the application to do**.

Before coding begins, developers gather and analyze customer needs.

### Example

Customer Requirement:

> "I need a food delivery app where users can order food and track delivery."

---

## Types of Requirements

### A. Functional Requirements

These describe **what the application should do**.

Examples:

* User Login
* Search Products
* Place Orders
* Online Payment

### B. Non-Functional Requirements

These describe **how well the application should perform**.

Examples:

* Security 🔒
* Fast Performance ⚡
* Scalability 📈
* User-Friendly Interface 😊

---

## Importance of Requirement Gathering

✅ Reduces misunderstandings
✅ Saves development cost
✅ Improves customer satisfaction
✅ Ensures project success

---

# 2️⃣ Development

## What is Development?

Development is the process of **writing code and building the application**.

Developers use programming languages such as:

* Java
* Python
* JavaScript
* C#

---

## Activities in Development

* Writing source code
* Designing databases
* Creating user interfaces (UI)
* Connecting frontend and backend
* Implementing business logic

### Example

For an E-commerce Application:

Developers create:

* Login Page
* Product Catalog
* Shopping Cart
* Payment Gateway

---

# 3️⃣ Peer Review

## What is Peer Review?

Peer Review is the process where **one developer reviews another developer's code**.

It is similar to checking a friend's homework before submission.

---

## Why is Peer Review Important?

✔ Detects bugs early
✔ Improves code quality
✔ Ensures coding standards
✔ Encourages knowledge sharing

### Example

* Developer A writes code.
* Developer B reviews the code.
* Issues are identified and fixed before testing.

---

# 4️⃣ Testing

## What is Testing?

Testing ensures that the application works correctly and is free from defects.

### Purpose of Testing

* Verify functionality
* Detect bugs
* Improve quality
* Ensure reliability

---

## Types of Testing

### A. Functional Testing

Checks whether features work correctly.

**Example:**

* Verify login functionality.

---

### B. Performance Testing

Checks application speed and scalability.

**Example:**

* Test whether the system supports 10,000 users simultaneously.

---

### C. Security Testing

Ensures user data is protected.

**Example:**

* Verify unauthorized users cannot access accounts.

---

### D. User Acceptance Testing (UAT)

Performed by customers to confirm that the application meets their requirements.
