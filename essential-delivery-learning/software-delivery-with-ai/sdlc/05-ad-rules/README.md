# 📘 Module 5: Application Development Roles

## 🎯 Learning Objectives

At the end of this module, you will be able to:

* Understand the various roles in an Application Development (AD) project.
* Identify the responsibilities of each team member.
* Understand how different roles collaborate to deliver software successfully.

---

# 🚀 Introduction

An Application Development project involves multiple people working together.

Think of developing software like building a house 🏠:

* Some people provide money 💰
* Some plan the work 📋
* Some design and build 🔨
* Some inspect quality ✅
* Some maintain infrastructure 🖥️

Each role has specific responsibilities.

---

# 👥 Application Development Roles

## 1️⃣ Business Sponsor

### Who are they?

The Business Sponsor is the person or organization that **funds the project** and approves major decisions.

### Responsibilities

* Provides budget 💰
* Defines business goals 🎯
* Approves project scope
* Reviews project progress

### Example

A bank wants a mobile banking app.

The bank management acts as the **Business Sponsor**.

---

## 2️⃣ Project Manager (PM)

### Who are they?

The Project Manager plans, coordinates, and manages the entire project.

### Responsibilities

* Creates project schedule 📅
* Assigns tasks
* Tracks progress
* Manages risks
* Ensures timely delivery

### Example

The PM ensures that developers, testers, and infrastructure teams work together smoothly.

---

## 3️⃣ Subject Matter Expert (SME)

### Who are they?

An SME is an expert in a particular domain or business area.

### Responsibilities

* Provides business knowledge 📚
* Clarifies requirements
* Helps solve domain-specific issues

### Example

For a banking application, a banking expert acts as an SME.

---

## 4️⃣ Business Lead

### Who are they?

The Business Lead represents business users and ensures the application meets business needs.

### Responsibilities

* Gathers requirements
* Communicates with stakeholders
* Prioritizes features
* Validates business processes

### Example

The Business Lead ensures that customer requirements are correctly implemented.

---

## 5️⃣ Technical Lead (Tech Lead)

### Who are they?

The Technical Lead guides the technical aspects of the project.

### Responsibilities

* Designs system architecture 🏗️
* Reviews code
* Solves technical challenges
* Guides developers

### Example

The Tech Lead decides which technologies to use.

---

## 6️⃣ Developer

### Who are they?

Developers write code and build the application.

### Responsibilities

* Write source code 💻
* Develop features
* Fix bugs
* Implement business logic

### Example

Developers create login pages, payment systems, and APIs.

---

## 7️⃣ Tester (QA Engineer)

### Who are they?

Testers verify that the application works correctly.

### Responsibilities

* Test application functionality ✅
* Report defects 🐞
* Perform regression testing
* Ensure software quality

### Example

Test whether login, payment, and search functions work correctly.

---

## 8️⃣ Infrastructure Team

### Who are they?

The Infrastructure Team manages servers, networks, and deployment environments.

### Responsibilities

* Set up servers 🖥️
* Configure networks 🌐
* Deploy applications 🚀
* Monitor system performance

### Example

They deploy the application to cloud platforms like AWS or Azure.
