# Module 2 – Agile

## 📌 Overview

This module introduces **Agile**, a modern software development methodology that focuses on flexibility, customer collaboration, and continuous improvement.

---

## 🎯 Learning Outcomes

At the end of this module, you will be able to:

* Describe Agile values and principles
* Differentiate Agile and Waterfall methodologies
* Summarize the benefits and challenges of Agile
* Identify when Agile is suitable for a project

---

# 1. What is Agile?

**Agile** is a way of developing software in **small steps** while continuously taking customer feedback and adapting to changes.

Instead of building the entire software at once, Agile develops software in **small iterations** called **Sprints**.

> **Agile welcomes change.**

---

# 2. Agile Workflow

```text
Plan
  ↓
Develop
  ↓
Test
  ↓
Review
  ↓
Release
  ↓
Repeat
```

Each cycle is called a **Sprint**.

### Sprint Duration

* Usually **1 to 4 weeks**
* Delivers a working feature

---

# 3. Real-World Example: Bus Booking App 🚌

Suppose you are building a **Bus Booking System**.

### Sprint 1

Build:

* User Registration
* Login

Customer Feedback:

> "Looks good!"

---

### Sprint 2

Build:

* Search Buses
* Filters

Customer Feedback:

> "Add AC/Non-AC filters."

Developers add the feature in the next sprint.

---

### Sprint 3

Build:

* Ticket Booking
* Online Payment

Customer Feedback:

> "Add UPI Payment."

Agile allows easy changes.

# 4. Agile Values

The Agile Manifesto has **4 core values**.

## 1. Individuals and Interactions over Processes and Tools

People and communication are more important than tools.

**Example:**

Talking to teammates solves problems faster than relying only on software tools.

---

## 2. Working Software over Comprehensive Documentation

Customers prefer software that works.

**Example:**

A working login page is more valuable than a 100-page design document.

---

## 3. Customer Collaboration over Contract Negotiation

Customers work with the development team throughout the project.

**Example:**

After every sprint, customers provide feedback.

---

## 4. Responding to Change over Following a Plan

Agile teams welcome changes.

**Example:**

If customers request dark mode, it can be added in future sprints.

---

# 5. Agile Principles

Some important Agile principles are:

* Deliver software frequently
* Welcome changing requirements
* Work closely with customers
* Build motivated teams
* Continuously improve
* Focus on technical excellence

---

# 6. Agile vs Waterfall

| Feature              | Waterfall      | Agile      |
| -------------------- | -------------- | ---------- |
| Development          | Sequential     | Iterative  |
| Requirement Changes  | Difficult      | Easy       |
| Customer Involvement | Low            | High       |
| Testing              | End of project | Continuous |
| Delivery             | One-time       | Frequent   |
| Flexibility          | Low            | High       |

---

## Waterfall Process

```text
Requirements
    ↓
Design
    ↓
Development
    ↓
Testing
    ↓
Deployment
```

## Agile Process

```text
Sprint 1 → Release
Sprint 2 → Release
Sprint 3 → Release
```

# 7. Benefits of Agile

✅ Faster delivery

✅ Better software quality

✅ Continuous customer feedback

✅ Easy to adapt to changes

✅ Reduced project risk

✅ Higher customer satisfaction

---

# 8. Challenges of Agile

❌ Requires active customer involvement

❌ Teams must communicate effectively

❌ Scope may increase frequently

❌ Planning can be difficult with changing requirements

---

# 9. When Should We Use Agile?

Agile is suitable when:

* Requirements change frequently
* Fast delivery is needed
* Customer feedback is important
* Teams collaborate closely

### Examples

* Mobile Applications
* E-commerce Websites
* Travel Booking Systems
* Food Delivery Apps
* SaaS Products

---

# 10. When Agile May Not Be Suitable

Agile may not be ideal for:

* Fixed requirements
* Heavy documentation projects
* Strict regulatory environments

### Examples

* Government Projects
* Defense Systems
* Safety-Critical Systems

---

# 🔑 Key Takeaways

* Agile develops software in small iterations called **Sprints**.
* Agile welcomes changing requirements.
* Customer feedback is collected regularly.
* Agile delivers software faster and continuously improves it.
* Agile is more flexible than Waterfall.
