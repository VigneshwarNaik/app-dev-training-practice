# Case Study 1: Air Travel Booking System

## 📌 Overview

This case study explains how **Lean Startup** and **Agile** can be used to develop a modern **Air Travel Booking System**.

The system allows users to:

* Search flights
* Book tickets
* Make online payments
* Track flight status

Instead of building everything at once, the team uses **MVP** and **Agile Sprints** to deliver features incrementally.

---

## 🎯 Learning Objectives

By the end of this case study, you will understand:

* How Lean Startup reduces risk
* What an MVP is
* How Agile develops software in Sprints
* How customer feedback improves products
* How teams handle changing requirements

---

# 1. The Problem

An airline company wants to build an online booking platform.

Required features:

* User Registration
* Flight Search
* Ticket Booking
* Online Payment
* Notifications
* Flight Tracking

Building all features at once is risky because customer requirements may change.

---

# 2. Lean Startup Approach

Lean Startup says:

> Build a small product first, learn from customers, and improve continuously.

---

## Build → Measure → Learn

```text
Build
  ↓
Measure
  ↓
Learn
  ↓
Repeat
```

---

# 3. Minimum Viable Product (MVP)

An **MVP** is the simplest version of a product that customers can use.

## Air Travel MVP

### Included Features

```text
✓ User Login
✓ Search Flights
✓ Book Tickets
✓ Payment
```

### Not Included Yet

```text
✗ Rewards Program
✗ AI Chatbot
✗ Seat Recommendations
✗ Multi-city Booking
```

The goal is to launch quickly and collect feedback.

---

# 4. Customer Feedback

After using the MVP, customers may say:

> "Please add UPI payment."

Developers then improve the product based on feedback.

This helps save:

✅ Time
✅ Money
✅ Effort

---

# 5. Agile Development

Agile develops software in small iterations called **Sprints**.

Each sprint usually lasts **1–4 weeks**.

---

## Sprint Plan

### Sprint 1

Build:

```text
✓ User Registration
✓ Login
```

---

### Sprint 2

Build:

```text
✓ Search Flights
✓ Flight Details
```

---

### Sprint 3

Build:

```text
✓ Ticket Booking
✓ Payment
```

---

### Sprint 4

Build:

```text
✓ Notifications
✓ Flight Tracking
```

After every sprint:

```text
Develop → Test → Review → Release
```

---

# 6. Key Agile Concepts

## Iterative Development

Software is built step by step.

Example:

```text
Sprint 1 → Login
Sprint 2 → Search
Sprint 3 → Payment
```

---

## Continuous Feedback

Customers review features regularly and provide suggestions.

---

## Collaboration

Team members work together:

* Developers
* Testers
* Designers
* Business Analysts

---

## Sprint Planning

The team decides which features to develop in the next sprint.

Example:

```text
Next Sprint:
✓ Notifications
✓ Flight Status
```

---

## Backlog Prioritization

A **Backlog** is a list of features.

### High Priority

```text
✓ Search Flights
✓ Booking
✓ Payment
```

### Low Priority

```text
✓ Rewards
✓ AI Chatbot
```

Important features are developed first.

---

## Delivering Incremental Value

After each sprint, customers receive working software.

Example:

* After Sprint 1: Users can login.
* After Sprint 2: Users can search flights.
* After Sprint 3: Users can book tickets.

---

# 7. Challenges Addressed

## Evolving Customer Requirements

Customers may request new features such as:

* UPI payments
* New filters
* Additional services

Agile makes it easy to adapt to changes.

---

## Real-Time Data Integration

The system connects with:

* Airline databases
* Payment gateways
* Notification services

---

## User Experience Enhancements

Improve:

* Search speed
* Mobile responsiveness
* Easy booking flow

---

# 8. Benefits of Lean Startup and Agile

✅ Faster delivery

✅ Early customer feedback

✅ Reduced project risk

✅ Better software quality

✅ Improved customer satisfaction

✅ Flexibility to adapt to change

# 🔑 Key Takeaways

* Lean Startup starts with an MVP.
* MVP contains only essential features.
* Agile develops software using Sprints.
* Customer feedback drives improvements.
* Features are delivered incrementally.
