# ☁️ What is Cloud Computing?

Cloud computing means delivering IT resources over the internet instead of using local servers or personal computers.

## Traditional On-Premise Model

In on-premise infrastructure, companies manage:

- Servers
- Storage
- Applications
- Databases

This requires:

- Hardware maintenance
- High cost
- Physical infrastructure

## Cloud Service Provider

In cloud computing:

- Resources are provided over the internet
- No need to maintain physical servers
- Users can access services anytime

Examples:

- AWS
- Microsoft Azure
- Google Cloud

---

# 🚀 How AWS Works

AWS provides cloud services through:

- AWS Management Console
- AWS CLI
- AWS SDKs

AWS offers many services such as:

| Category | Examples |
|---|---|
| Compute | EC2, Lambda |
| Storage | S3 |
| Database | RDS, DynamoDB |
| Networking | VPC |
| Security | IAM |
| Analytics | Athena |
| Machine Learning | SageMaker |

# 🧩 Cloud Computing Models

## 1. SaaS (Software as a Service)

Users directly use software through the internet.

### Examples
- Gmail
- Google Docs
- Zoom

### User Access
- Only uses the application
- No control over infrastructure

---

## 2. PaaS (Platform as a Service)

Provides a platform for developers to build applications.

### Examples
- AWS Elastic Beanstalk
- Heroku

### User Access
- Manage applications and data
- Cloud provider manages servers and OS

---

## 3. IaaS (Infrastructure as a Service)

Provides servers, storage, and networking.

### Examples
- AWS EC2
- Microsoft Azure VM

### User Access
- Full access to OS and applications

---

# 🌐 Cloud Deployment Models

## 1. On-Premises / Private Cloud

- Infrastructure owned by one organization
- More control and security
- Higher maintenance cost

---

## 2. Public Cloud

- Services available over internet
- Shared infrastructure
- Cost-effective and scalable

Example:
- AWS Public Cloud

---

## 3. Hybrid Cloud

Combination of:

- On-premise infrastructure
- Public cloud services

Benefits:
- Flexibility
- Better scalability

---

# 🌎 AWS Global Infrastructure

AWS infrastructure consists of:

| Component | Description |
|---|---|
| Regions | Geographic locations |
| Availability Zones | Multiple isolated datacenters |
| Edge Locations | Content delivery locations |

---

# 📍 Region

A Region is a physical geographic area containing multiple Availability Zones.

### Examples
- Mumbai
- Singapore
- US East

### Selecting a Region

Factors to consider:

- Data governance
- Legal requirements
- Proximity to customers
- Service availability
- Cost

---

# 🏢 Availability Zone (AZ)

An Availability Zone is:

- One or more isolated datacenters inside a region

Benefits:

- High availability
- Fault tolerance
- Backup support

Example:

- ap-southeast-1a
- ap-southeast-1b
- ap-southeast-1c

---

# 🌐 Edge Locations

Edge locations help deliver content faster to users.

Used in:

- AWS CloudFront
- CDN services

Benefits:

- Reduced latency
- Faster content delivery
